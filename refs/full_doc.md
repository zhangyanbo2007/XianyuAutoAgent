# Claude Code采购用途及费用

### 1. 背景与目的

本采购的背景和目的是服务于记忆系统插件项目和应急两个子项目的 Claude Code vibecoding 开发，以及日常开发文档管理用途。现状是采用阿里百炼云平台的资源（qwen3.6-plus、deepseek-v4-pro、glm-5.1 等模型），目前公司只有 API 消费模式而无 code plan 模式，每天消耗大量 token，非常不划算。另外模型性能低也会导致简单的任务需要迭代很多轮才能成功完成，这无形中进一步消耗了大量 token。基于上述两个项目，目前平均每天几千万 token 的消耗（大部分 qwen3.6-plus，少部分 glm-5.1），成本较高。购买 Claude Code 订阅预计会有较大改善。

实测依据：应急项目已开发 1 周，试用 Claude Code Pro 版本期间，基本每天宕停 2 次，整体不够使用。两个项目并行开发时 Pro 版本配额严重不足，考虑后续使用余量，真实使用场景需 5x 量级。

本采购服务于两个子项目：

1. 应急项目：Claude Code vibecoding 开发（设计文档编写；代码编写；代码分析；代码检查与优化；测试评估）
2. 智能体记忆系统插件项目：Claude Code vibecoding 开发（设计文档编写；代码编写；代码分析；代码检查与优化；测试评估）

### 2. 采购工具价值与必要性

#### 2.1 效率提升显著（缩短迭代周期）

在现有流程中，人工对日志与 badcase 的逐条分析、归因与改写 prompt，完成一次"可复用的详细结论 + 优化方案"通常至少需要 1 个工作日。
引入 Claude Code 后，可在更短时间内完成结构化分析与可落地改写建议，显著提高迭代频率，支持"日更多版"的持续优化节奏，整体效率提升明显。

#### 2.2 分析更系统、更准确（支撑下一步优化决策）

Claude Code 能够对 badcase、feedback、流程图（flowchart）、节点（nodes）、约束策略等内容进行系统化对比与归因，输出更聚焦于"错误机制"的结论，减少主观判断偏差，从而更有效地指导下一轮优化方向。

Codex 与 Claude Code 定位互补，组合使用可在「分析可信度」与「需求落地效率」两方面进一步增强：

### 3. 现状问题与改进方向（示例摘录）

#### 3.1 Feedback 分析存在的问题

**问题 1：错误分类粒度过粗，难以定位真实原因**
典型输出中，错误类型与严重级别集中在少数类别（如 "COT缺失""P0-致命"），导致：

- 早期确有 COT 缺失，但后期 COT 已建立后，错误机理发生变化却未被有效区分；

- 缺乏对数学推理错误的细分（如概念理解错误、策略选择错误、计算错误等），难以形成针对性的优化闭环。

**问题 2：建议缺乏可执行性，无法直接转化为改写动作**
常见建议停留在"补充知识"层面（例如"补充组合计数原理"），但：

- "补充知识"不等于"解决问题"，缺少针对具体错误的精准补充；

- 对单个 badcase 的关键陷阱缺少明确指向。例如三角形计数类问题，核心往往是"共线点计数中的顶点归属/重复计数处理"，而非泛泛的组合公式复习。
Claude Code 可进一步把"笼统建议"拆解为"可直接写入 prompt 的规则/检查项/策略步骤"，提升方案落地效率。

---

#### 3.2 Proposer（流程图/节点约束设计）存在的问题

**问题 1：流程复杂度上升，但收益边际递减**
从早期版本到后期版本，流程图节点与分支显著增加（8 节点 → 15+ 节点），但准确率提升有限（例如 30% → 40%）。
这意味着：

- 过度细化会增加模型的决策负担与执行难度；

- "约束越多、流程越细"并不必然带来"推理越准确"。
Claude Code 可辅助进行结构裁剪与关键节点提纯，以更少的复杂度获得更稳定的效果增益。

**问题 2：优化重心偏"格式约束"，弱化"能力约束"**
当前约束中大量限制模型"不能做什么"（例如行数限制、禁止推导、禁止引入变量等），但缺少对"如何避免典型数学错误"的正向指导。
Claude Code 能帮助将优化重心从"输出形态合规"转向"推理质量提升"，如：错误检查策略、常见陷阱清单、验证步骤模板等。

**问题 3：知识补充偏通用，缺乏针对性**
现有补充多为教科书级通用知识（组合公式、勾股/余弦定理等），模型本身已具备；真正需要的是"针对具体错误模式的专项知识与规则化检查"。
Claude Code 可结合 badcase 形成"专项知识片段 + 可复用规则"，更贴合业务问题形态。

### 4. 使用计划与产出预期（可量化）

#### 4.1 使用内容

- 应急项目 vibecoding 开发：设计文档编写；代码编写；代码分析；代码检查与优化；测试评估
- 智能体记忆系统插件 vibecoding 开发：设计文档编写；代码编写；代码分析；代码检查与优化；测试评估
- GEPA / 训练日志分析：定位当前优化瓶颈与主要误差模式
- Prompt 优化与版本迭代：基于问题归因给出可执行改写，并形成可复用模板
- badcase 结构化归因：沉淀"错误类型 → 检查策略 → Prompt修正"闭环，提高后续优化复用率

#### 4.2 预期迭代节奏（人天与版本数）

- 单次"COT 分析 + Prompt 优化"：约 1.2（claude code Pro Max 5x 使用额度）/次
- 每日优化版本数：约 4.8（claude code Pro Max 5x 使用额度）/日（两人协作，上下班各一次迭代）
  - 上班：分析夜间实验结果并产出 2 版优化
  - 下班：分析白天实验结果并产出 2 版优化

通过提高单位时间迭代次数，可更快收敛到稳定策略，并减少"低效试错"的时间成本。
