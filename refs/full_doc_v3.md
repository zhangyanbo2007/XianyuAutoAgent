# Claude Code 采购申请

### 1. 背景与目的

本采购服务于两个 AI 项目的 Claude Code vibecoding 开发：

1. **应急指挥 AI 系统**（emergency_command + duty_processor）：将应急指挥中心的电话录音自动转化为结构化的应急事件报告。包含指挥中心事件处理和日常值班事件处理两条独立流水线，涉及 ASR 语音转写、LLM 事件判定与信息抽取、FAISS 语义匹配等多模块串联，技术栈为 Python 3.11 + Nanobot Agent 框架 + FastAPI + DashScope（通义千问）。

2. **主动智能体记忆系统插件**（Proactive Agent Memory Core Plugin）：为 AI 智能体提供类人脑的记忆系统，解决"金鱼记忆困境"（跨会话记忆混乱缺失）和"间歇式主场困境"（无主动知识挖掘能力）。基于认知科学理论设计了 7 类记忆维度、4 级链路存储、6 类网络互联、3 级巩固机制，技术栈为 Python，当前处于架构设计向代码实现转化的关键阶段。

现状是两个项目均采用阿里百炼云平台的资源（qwen3.6-plus、deepseek-v4-pro、glm-5.1 等模型），公司只有 API 消费模式而无 code plan 模式，每天消耗大量 token，非常不划算。模型性能低也导致简单任务需要迭代很多轮才能完成，进一步消耗大量 token。目前两个项目平均每天几千万 token 的消耗（大部分 qwen3.6-plus，少部分 glm-5.1），成本较高。购买 Claude Code 订阅预计会有较大改善。

实测依据：应急项目已开发 1 周，试用 Claude Code Pro 版本期间，基本每天宕停 2 次，整体不够使用；Codex Pro 版本同样每天宕停 2-3 次，无法满足持续开发需求。两个工具的 Pro 版本配额均严重不足，考虑后续使用余量，真实使用场景均需 5x 量级。

### 2. 采购工具价值与必要性

#### 2.1 应急指挥 AI 系统的开发需求

该系统包含两条核心流水线：

- **指挥中心事件处理**（emergency_command）：ASR 转写 → 应急事件判定 → 结构化信息抽取（12 项字段）→ 补充判定（事件类型分类 / 响应等级评估 / 敏感事件判定）→ 组装最终 JSON。事件类型分类使用 FAISS + BGE-large-zh-v1.5 向量语义匹配，响应等级按伤亡人数分 I-IV 级。

- **日常值班事件处理**（duty_processor）：ASR 转写+规范化 → 事件类型分类 → 名称摘要生成 → 接续判断（是否为历史事件延续）→ 组装结果。事件分类覆盖日常咨询、院内医疗急救、应急指挥紧急医学救援等场景。

Claude Code 在该项目中的主要用途：

- 流水线各环节的 Prompt 工程迭代（事件判定、信息抽取、分类、敏感事件判断等）
- badcase 分析与 Prompt 优化，沉淀"错误类型 → 检查策略 → Prompt 修正"闭环
- FAISS 向量检索模块的调优与评测框架维护
- Nanobot Agent 框架下的 Skill / Tool 编排与调试
- 接口文档与技术方案的自动化生成

#### 2.2 智能体记忆系统插件的开发需求

该项目基于认知科学理论，设计了完整的记忆架构：

- **7 类记忆维度**（M1-M7）：语义记忆（智能体画像/用户画像/世界知识/元认知）、情景记忆（任务轨迹/对话轨迹）、非陈述性记忆（程序性技能/触发技能/情绪技能/习惯）
- **4 级链路存储**：内容标题 → 摘要 → 完整内容 → 向量编码
- **6 类网络互联**：层级关联、社区聚类、因果、概念、实体、语义关联
- **3 级巩固机制**：突触巩固（分钟级）、系统巩固（天级）、睡眠巩固（记忆重放与重构）
- **主动任务生成**：基于预测编码理论的主动对话任务与主动世界任务

Claude Code 在该项目中的主要用途：

- 从架构设计文档到代码实现的 vibecoding 全流程
- 记忆存储、检索、巩固、任务生成等核心模块的代码编写与测试
- LongMemEval 和 PersonaMem 基准评测框架搭建
- 论文 / 专利的技术文档撰写

#### 2.3 效率提升显著

在现有流程中，人工对日志与 badcase 的逐条分析、归因与改写 prompt，完成一次"可复用的详细结论 + 优化方案"通常至少需要 1 个工作日。引入 Claude Code 后，可在更短时间内完成结构化分析与可落地改写建议，显著提高迭代频率，支持"日更多版"的持续优化节奏。

#### 2.4 Codex 代码审查与交叉验证互补

Codex 与 Claude Code 定位互补，Codex 主要承担以下职责：

- **代码审查**：对 Claude Code 生成的代码进行独立审查，发现逻辑漏洞、边界问题和潜在风险
- **交叉验证**：针对同一 badcase 或 Prompt 改写方案，两者独立分析、相互印证，减少单一模型的判断偏差
- **局部代码优化**：针对特定模块的性能瓶颈或代码质量问题，进行精准优化

两者组合使用可在"代码质量"与"开发效率"两方面进一步增强，由 Codex 负责审查与验证、Claude Code 负责高效执行，扬长避短。
