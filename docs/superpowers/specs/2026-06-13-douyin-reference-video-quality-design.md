# 抖音剪辑参考视频质量优化设计

## 背景

项目位于 `projects/douyin-jianji`。目标是根据 `demand/` 中的 Excel 需求生成视频，成片在内容组织和表现形式上对齐 `reference_videos/` 中的参考视频。

当前项目已经有一套 Python pipeline，但现有产物更接近“静态幻灯片 + TTS + 底部字幕”。参考视频的质量主要来自高密度政策解读脚本、横屏信息图页面、真实/拟真素材截图、顶部栏目标题、底部字幕条、频繁换页和统一的视觉系统。因此优化重点不是单纯提高分辨率，而是重建从需求解析到成片质量检查的生成链路。

## 已确认方向

采用严格参考视频风格的横屏信息图方案。

- 输出以横屏信息图视频为主，画面比例和构图对齐参考视频。
- `demand/douyinwenanjiaoben.xlsx` 生成 1 条视频。
- `demand/618.xlsx` 识别同一工作表中的 3 个脚本块，生成 3 条视频。
- 总计至少生成 4 条需求视频。
- 参考视频用于定义内容密度、版式、字幕和剪辑形式，不要求照搬原视频内容。

## 成功标准

1. `demand/` 中每个脚本块都能被独立解析、命名、生成脚本 JSON、分镜、音频、字幕和视频。
2. 生成视频不再出现 `项目1`、`项目2`、`【画面】`、`【字幕】` 等解析残留。
3. 每条视频至少包含封面/开场、核心解释、数据或流程信息图、总结/CTA 等镜头类型。
4. 画面具备参考视频的固定顶部标题栏、底部字幕栏、中间信息图主体和统一水印/品牌弱标识能力。
5. 字幕为烧录字幕，位置、字号、描边和背景条接近参考视频底部字幕效果。
6. 音频输出优先使用高质量 TTS，最终合成音频为 AAC 44.1kHz；TTS 或 BGM 降级时在质量报告中明确记录。
7. 视频输出为 30fps，使用可复查的 ffmpeg 参数，并生成每条视频的质量报告。
8. 自动化测试覆盖 Excel 多脚本解析、视觉文本清洗、模板选择、字幕生成和质量报告结构。

## 参考视频风格拆解

参考视频是横屏信息图视频，不是竖屏口播视频。可复刻的稳定元素如下：

- 顶部蓝绿色标题栏，显示栏目标题，如“618光伏新政解析”或“山东光伏新政策解读”。
- 底部蓝绿色字幕条，字幕大号白字，配合口播逐段出现。
- 中间为主视觉区，常见深色科技背景或浅色政策解读页。
- 主视觉区由大标题、政策截图/照片、数据卡片、流程卡片、三色分区卡片组成。
- 每 3 到 8 秒切换一个页面或页面状态，信息密度明显高于现有静态占位图。
- 配色以深青蓝、灰蓝、白色、绿色强调、黄色/红色风险提示为主。

## 架构

整体链路拆成六个清晰阶段：

```text
demand Excel
  -> 脚本块解析
  -> 分镜规划
  -> 信息图页面渲染
  -> TTS 与字幕
  -> 视频合成
  -> 质量报告
```

### 1. 脚本块解析

解析器需要从 Excel 中识别多个完整脚本块。一个脚本块从“抖音短视频脚本”行开始，到下一个“抖音短视频脚本”行或工作表结尾结束。

每个脚本块输出结构化数据：

- `source_file`
- `sheet_name`
- `block_index`
- `program_title`
- `headline`
- `hashtags`
- `style`
- `duration_text`
- `sections`
- `execution_tips`

每个 section 保留：

- 时间范围和秒数
- 标签，如“痛点/悬念”“核心干货输出”
- 清洗后的画面字幕
- 口播文本
- 素材建议
- 推断出的镜头模板

清洗规则必须删除 Excel 中用于说明的标记，例如 `【画面】`、`【字幕】`，并移除不适合渲染的 emoji。流程和列表需要从视觉文案中提取出干净条目，例如 `备案 -> 施工 -> 并网`。

### 2. 分镜规划

现有脚本段落有时过长，尤其 `618.xlsx` 的核心干货段通常为 15 秒左右。分镜规划器需要把段落转换为多个视觉镜头，但不改写原始口播事实。

规则：

- 0 到 3 秒的悬念段生成强标题开场。
- 8 秒以上的长段按句号、分号、顿号或信息点拆成多个镜头。
- 含数字和政策结论的内容优先生成数据大字镜头。
- 含 `绿/黄/红`、百分比或区间的内容生成三色分区镜头。
- 含 `流程`、`顺序`、`步骤`、`备案`、`并网` 的内容生成流程镜头。
- 结尾引导生成 CTA 镜头。

每条 28 到 32 秒视频建议生成 6 到 9 个镜头，避免现有 4 到 5 张静态图拖太久。

### 3. 信息图页面渲染

渲染器应从“幻灯片式空卡片”升级为“参考视频式信息图页面”。核心模板：

- `cover_dark`：深色封面，大标题、绿色短横线、年份或政策关键词。
- `headline_warning`：痛点/风险大字，突出“白装”“拒绝并网”“红线取消”等。
- `data_release`：大数字和解释卡片，例如 `50GW`、`80%`、`3个工作日`。
- `process_flow`：横向或阶梯流程图，例如 `备案 -> 施工 -> 并网`。
- `zone_cards`：绿/黄/红三色分区规则。
- `policy_explain`：左侧政策/现场图占位，右侧三条解读卡。
- `material_grid`：资料清单卡片。
- `channel_steps`：线上/线下办理渠道 + 后续流程。
- `cta_summary`：总结和评论区引导。

模板需要支持真实素材缺失时的高质量占位：政策文件样式卡、国家电网办公场景示意、光伏屋顶示意、储能/电网图标。占位不能是空卡片。

### 4. 素材策略

优先使用本地和可生成素材：

- 从参考视频抽取的风格参数只用于模板，不直接搬运参考视频画面。
- 使用代码绘制政策文件页、流程图、三色分区和数据卡片。
- 可选调用 AI 图片生成作为背景或照片占位，但 pipeline 必须在无外部图片 API 时仍能生成可接受的信息图视频。
- 每条视频输出 `storyboard.json`，记录每个镜头使用的模板、文案和素材来源。

### 5. 音频与字幕

TTS 优先级：

1. MiMo TTS。
2. edge-tts。
3. 已有本地音频缓存。
4. 失败时生成无口播视频并在质量报告中标记为不达标。

字幕策略：

- 根据 TTS 实际时长生成字幕时间轴。
- 长句拆成短字幕，底部每条控制在参考视频类似的单行或双行长度。
- ASS 字幕使用底部居中、白字、黑描边、轻微阴影、蓝绿色底栏。
- 字幕文本不能超过画面宽度，过长需自动换行或拆段。

### 6. 视频合成

合成器职责：

- 逐镜头生成视频片段。
- 为静态信息图添加轻微缩放、推拉或淡入淡出，避免完全静止。
- 合并 TTS、BGM、字幕和镜头片段。
- 输出 30fps 横屏 MP4。
- 音频采样率统一到 44.1kHz。
- 为每条视频保留关键帧截图，便于人工快速检查。

## 输出结构

建议输出到 `output/<slug>/`：

```text
output/<slug>/
  script.json
  storyboard.json
  narration.mp3
  narration.srt
  narration.ass
  slides/
  frames/
  quality_report.json
  quality_report.md
  <slug>.mp4
```

`slug` 从脚本标题生成，重复时追加块编号。

## 质量报告

每条视频生成 `quality_report.md` 和 `quality_report.json`。至少包含：

- 输入 Excel、sheet、block index。
- 视频参数：时长、分辨率、帧率、视频编码、码率。
- 音频参数：采样率、声道、编码、是否使用 TTS 降级。
- 镜头数量、平均镜头时长、模板覆盖情况。
- 字幕条数、最长字幕字符数。
- 自动检查结果：是否有解析残留、是否有空占位、是否缺少音频、是否缺少字幕。
- 与参考风格的检查项：顶部标题栏、底部字幕栏、信息图主体、数据/流程/CTA 覆盖。

质量报告不是最终审美判断，但必须能拦住明显低质产物。

## 测试策略

需要先补自动化测试，再改实现。

测试重点：

- `demand/618.xlsx` 解析出 3 个脚本块。
- `demand/douyinwenanjiaoben.xlsx` 解析出 1 个脚本块。
- 清洗后的视觉文案不包含 `【画面】`、`【字幕】` 或 emoji。
- 结尾引导被识别为 CTA，而不是普通网格。
- 三色分区内容被识别为 `zone_cards`。
- `50GW`、`80%` 等数字内容被识别为数据镜头。
- 质量报告能发现空占位、解析残留和缺失音频。
- 渲染出的关键帧非空，且包含顶部栏和底部字幕栏。

## 非目标

- 不做真人出镜。
- 不构建剪映工程文件。
- 不保证政策事实自动联网更新；脚本事实以 `demand/` 为准。
- 不把参考视频原画面直接混入生成视频。
- 不在本阶段实现竖屏版本。

## 实施边界

本次优化应集中在 `projects/douyin-jianji/pipeline/` 及其测试、文档和输出报告。已有的非本项目改动不得回滚。实现时要保持 pipeline 可离线运行到“信息图视频 + 质量报告”，即使外部 TTS 或图片 API 不可用，也要给出明确降级状态。
