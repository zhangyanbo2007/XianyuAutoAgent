# 日报格式模板

**何时阅读**：Step 5 生成日报时读取此文件，确认每条内容的格式规范。

---

## 标题层级规则

- `##` 板块主节（如"二、大模型板块 🤖")
- `###` 子主题分组（如"后训练与对齐"、"RL与推理优化"、"Agent安全与对抗"）
- `####` 单个条目（论文/资讯标题）

## 每条论文格式（学术板块）

arXiv 论文：
```markdown
#### 中文解读（英文原标题）

> 🏠 **来源：arXiv 2605.16289 | Chen et al., ByteDance** | 2026-05-16 | https://arxiv.org/abs/2605.16289

![架构图](05-images/2605.16289-x1.png)

**摘要**：2-3句实质性中文解读...
```

期刊论文：
```markdown
#### 中文解读（英文原标题）

> 🏠 **来源：Nature Neuroscience | Park et al., MIT** | 2026-05-10 | https://doi.org/xxx

**摘要**：2-3句实质性中文解读...
```

## 每条资讯格式（news/builders板块）

```markdown
#### 新闻标题

> 🏠 **来源：Anthropic 官方博客** | 2026-04-23 | https://anthropic.com/news/xxx

![官方配图](05-images/xxx.png)

**摘要**：2-3句实质性中文解读...（资讯板块也可将中文解读直接写在来源行后，无**摘要**：前缀）
```

X/Twitter 推文：
```markdown
#### 推文标题

> 🏠 **来源：Sam Altman（OpenAI CEO）** | 2026-05-10 | https://x.com/sama/status/xxx

**摘要**：2-3句实质性中文解读...
```

YouTube 视频：
```markdown
#### 视频标题

> 🏠 **来源：YC Startup School** | 2026-05-21 | https://www.youtube.com/watch?v=xxx

![缩略图](05-images/yt-xxx.jpg)

**摘要**：2-3句实质性中文解读...
```

Podcast：
```markdown
#### 播客标题

> 🏠 **来源：Training Data（AI前沿对话节目）** | 2026-05-08 | https://xxx

**摘要**：2-3句实质性中文解读...
```

## 来源行日期要求

日期是信息溯源的关键要素，没有日期读者无法验证事件时效性。所有来源必须标注发布日期，格式统一为 `YYYY-MM-DD`。

## 板块洞察格式

每个主板块（🤖/🧠/🔬等）末尾附板块洞察：

```markdown
> 💡 **板块洞察：** 本板块论文的核心趋势和关联，2-3句总结，作为狐狸点评的铺垫。
```

## 狐狸点评格式

```markdown
## 三/五/六、狐狸点评 🦊

本日N篇XXX论文的三条主线交汇于一个核心趋势：...

1. **主线1** — ...
2. **主线2** — ...
3. **主线3** — ...

**小王子的XXX视角**：...（与kb/memory中的核心认知关联）
```
