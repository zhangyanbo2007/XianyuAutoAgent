核心要点

问题/背景
STAR 针对微服务 RCA agents 的关键可靠性问题：早期证据收集、假设构造或因果分析错误会沿 reasoning trace 传播，最终破坏诊断。
方法/机制
它把 RCA workflow 拆成 Evidence Package、Hypothesis Set、Analysis Structure 和 Decision Report 四个阶段，并把 agent 失败视为可定位、可修复的 stage-local bug。
结果/证据
它值得正式收录，因为它提供了 agent observability/repair 的可复用模式：stage-wise auditing、fast/slow routing、counterfactual stage localization 和 patch-and-replay。
收录价值
它没有更高，是因为目前领域集中在 AIOps/RCA，能否推广到通用 coding agents、research agents 和业务 workflow 还需验证。
完整收录解读

STAR 针对微服务 RCA agents 的关键可靠性问题：早期证据收集、假设构造或因果分析错误会沿 reasoning trace 传播，最终破坏诊断。

它把 RCA workflow 拆成 Evidence Package、Hypothesis Set、Analysis Structure 和 Decision Report 四个阶段，并把 agent 失败视为可定位、可修复的 stage-local bug。

它值得正式收录，因为它提供了 agent observability/repair 的可复用模式：stage-wise auditing、fast/slow routing、counterfactual stage localization 和 patch-and-replay。

它没有更高，是因为目前领域集中在 AIOps/RCA，能否推广到通用 coding agents、research agents 和业务 workflow 还需验证。

原始摘要与中文对照

中文对照翻译

STAR 通过将 LLM 根因分析代理的跟踪中的故障定位到工作流阶段，并应用特定于阶段的补丁和重放修复，来修复 LLM 根因分析代理的跟踪。

原始摘要

STAR repairs LLM-based root-cause-analysis agent traces by localizing failures to workflow stages and applying stage-specific patch-and-replay repair with counterfactual evaluation.
