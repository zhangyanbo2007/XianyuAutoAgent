中文对照翻译

今天，计算机使用代理（CUAs）主要部署为单串代理。这种设置对于需要任务分解、并行执行和基于新信息的一致重新规划的复杂长时限任务来说是不理想的。在本文中，我们认为我们应该转向评估和构建多代理计算机使用（MACU）系统。这些系统，强调规划和并行执行，消除了单代理CUAs的许多缺点。我们提出了一种通用的多代理设置，其中一个管理器模型将计算机使用任务分解为由子任务组成的有向无环图（DAG），编码了子代理的相关依赖关系和目标。在每次迭代中，管理器将并行CUA子代理分配到DAG的已准备好前沿的节点上，并在从子代理获得新发现时不断修改DAG（添加、取消或重写节点）。这种设计将计算机使用的部分可观测环境视为一个首要挑战：下游代理可能无法重新观察的信息被保留并通过管理器和DAG结构传递。我们在桌面（OSWorld）和网络导航（Online-Mind2Web、WebTailBench、Odysseys）基准上，通过MACU在强单代理基线之上持续改进3.4-25.5%，表现出更友好的测试时间计算缩放，并解决了单代理CUAs陷入困境的复杂长时限任务。在Odysseys，一个长时限网络导航基准上，MACU还使平均任务完成时钟时间提高了约1.5倍，证明了其加速传统缓慢的CUA管道的有效性。我们的研究表明，多代理协调是扩展计算机使用代理以更有效地更长时间地工作的一个有希望的方向。所有代码和交互式可视化均可在https://jykoh.com/multi-agent-computer-use上发布。

原始摘要

Computer use agents (CUAs) today are primarily deployed as single serial agents. This setup is suboptimal for complex long-horizon tasks that benefit from task decomposition, parallel execution, and consistent re-planning based on new information. In this paper, we argue that we should instead move towards evaluating and building multi-agent computer use (MACU) systems. These systems, which emphasize planning and parallel execution, alleviate many of the shortcomings of single-agent CUAs. We propose a general multi-agent setup in which a manager model decomposes computer use tasks as a directed acyclic graph (DAG) of subtasks, encoding relevant dependencies and goals for subagents. At each iteration, the manager dispatches parallel CUA subagents to carry out nodes on the ready frontier of the DAG, and continuously revises the DAG (adding, canceling, or rewriting nodes) as new findings arrive from subagents. This design treats the partially observable environment of computer use as a first class challenge: information that downstream agents may not be able to re-observe are retained and passed forward through the manager and DAG structure. We demonstrate that MACU consistently improves over strong single-agent baselines by 3.4 − 25.5% on desktop (OSWorld) and web navigation (Online-Mind2Web, WebTailBench, Odysseys) benchmarks, exhibits more favorable test-time compute scaling, and solves complex long-horizon tasks where single-agent CUAs get stuck. On Odysseys, a long-horizon web navigation benchmark, MACU also improves average task completion wall-clock time by ∼1.5×, demonstrating its efficacy in speeding up traditionally slow CUA pipelines. Our findings highlight that multiagent coordination is a promising axis for scaling computer use agents to work productively for longer and more effectively. We release all code and interactive visualizations at https://jykoh.com/multi-agent-computer-use.
