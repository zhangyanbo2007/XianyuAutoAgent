核心要点

问题/背景
这篇 benchmark 的价值在于它把 exploratory data analysis 当成 process problem，而不是 final-answer problem。现实里的数据分析经常发生在数据脏、先验弱、目标不完全清晰的环境里，单看最终答案会掩盖大量过程差异。
方法/机制
DataClaw 提供的不是单纯更大的数据集，而是带 milestone annotation 的 process-oriented evaluation。这样不仅能判断 agent 成没成功，还能看它卡在哪一步、用了什么探索策略、是不是出现了 reasoning collapse。
结果/证据
它值得正式收录，因为企业分析、政策研究和复杂 BI workflow 都会遇到类似结构。这个 benchmark 的外溢点不在具体数据域，而在把探索式数据工作正式写成 agent evaluation interface。
收录价值
它没有更高，是因为当前场景仍主要集中在 data-analysis agents，而不是更广泛的 general-purpose coworker agents。它的 process evaluation 设计很强，但还不是统一上位 benchmark。

中文对照翻译

DataClaw：一个面向过程的探索性真实世界数据分析智能体基准。在行业监测、政策影响分析和咨询研究等场景中，分析师通常从一个具体问题开始，但相关证据并非直接给出。他们必须检查数据环境，识别有用来源，对齐跨来源信息，处理数据噪声，并综合得出结论。随着LLM智能体变得更加自主，评估这种探索性过程变得越来越重要。现有基准在评估数据分析能力方面已取得显著进展。最近面向智能体的基准（Gu et al., 2024; Huang et al., 2024; Bigeard et al., 2025; Egg et al., 2025）进一步引入了工具使用和多步分析推理。然而，这些尚未完全符合探索性真实世界数据分析的要求。我们发现了两个关键差距。❶ 先验引导的数据设置。许多基准（Lei et al., 2024; Hu et al., 2024; Huang et al., 2024; Egg et al., 2025; Wang et al., 2025; Bigeard et al., 2025）在先验引导的数据设置中评估智能体。相关来源、模式或数据噪声条件通常部分指定。这降低了智能体发现证据、对齐跨来源信息和处理数据噪声的需求。然而，在复杂的真实世界分析中，此类数据先验通常不可用或不完整。因此，这些基准未能充分测试自主数据分析所需的探索性过程。为了明确区分，我们在表1中比较了代表性基准在探索性数据分析所需关键属性方面的表现。❷ 面向结果的黑盒评估。现有工作（Gupta et al., 2026; O’Leary, 2026; Ding et al., 2026; Ye et al., 2026）主要采用最终答案准确性作为评估标准。虽然这足以对整体模型性能进行排名，但它无法评估中间步骤的正确性或归因失败的根本原因。如果没有过程级诊断，评估结论将无法提供可操作的指导，这削弱了对自主数据分析智能体的评估，而这种评估需要测试它们在未充分探索的数据环境中执行探索性分析的能力。然而，许多现有基准在先验引导的数据设置中强调最终答案的准确性，并且对推理过程评估的支持有限。我们引入了DataClaw，一个面向过程的探索性真实世界数据分析基准。DataClaw包含约206万条来自企业、行业和政策领域的真实世界记录，并保留了原始数据噪声。它进一步包含了492个源自智库咨询场景的跨领域任务，每个任务都标注了中间里程碑，用于过程级评估。这些标注使得DataClaw能够衡量智能体进展到何种程度以及其推理在何处中断。对八个先进LLM进行的实验表明，当前智能体在此设置下远未达到可靠水平，其中七个模型的总体准确率低于50%。过程分析进一步揭示了错误答案背后隐藏的部分进展以及不同模型之间独特的探索策略。总之，DataClaw为探测自主数据分析智能体的能力边界提供了一个数据约束较少的诊断测试平台1。

They must inspect data environments, identify useful sources, align information across sources, handle data noise, and synthesize a conclusion. As LLM agents become more autonomous, evaluating this exploratory process becomes increasingly important. Existing benchmarks have made substantial progress in evaluating data analysis capabilities. Recent agent-oriented benchmarks (Gu et al., 2024; Huang et al., 2024; Bigeard et al., 2025; Egg et al., 2025) further introduce tool-use and multi-step analytical reasoning. However, these are not yet fully aligned with the requirements of exploratory realworld data analysis. We identify two key gaps. ❶ prior-guided data settings. Many benchmarks (Lei et al., 2024; Hu et al., 2024; Huang et al., 2024; Egg et al., 2025; Wang et al., 2025; Bigeard et al., 2025) evaluate agents in prior-guided data settings. Relevant sources, schemas, or data noise conditions are often partly specified. This reduces the need for agents to discover evidence, align information across sources, and handle data noise. Yet in complex real-world analyses, such data priors are often unavailable or incomplete. As a result, these benchmarks do not fully test the exploratory process autonomous data analysis requires. To make this distinction explicit, we compare representative benchmarks along the key properties required for exploratory data analysis in Table 1. ❷ Outcome-oriented black-box evaluation. Existing work (Gupta et al., 2026; O’Leary, 2026; Ding et al., 2026; Ye et al., 2026) predominantly adopts final-answer accuracy as the evaluation criterion. While this suffices to rank overall model performance, it cannot assess the correctness of intermediate steps or attribute the root causes of failures. Without process-level diagnostics, evaluation conclusions remain non-actionable, undermining Evaluating autonomous data analysis agents requires testing their ability to perform exploratory analysis in underexplored data environments. However, many existing benchmarks emphasize final answer accuracy in priorguided data settings and provide limited support for reasoning process evaluation. We introduce DataClaw, a process-oriented benchmark for exploratory real-world data analysis. DataClaw contains approximately 2.06 million real-world records across enterprise, industry and policy domains, with native data noise preserved. It further includes 492 crossdomain tasks derived from think-tank consulting scenarios, each annotated with intermediate milestones for process-level evaluation. These annotations allow DataClaw to measure how far an agent progresses and where its reasoning breaks down. Experiments with eight advanced LLMs show that current agents remain far from reliable in this setting, with seven models achieving below 50% overall accuracy. Process analysis further reveals partial progress hidden behind wrong answers and distinct exploration strategies across models. Overall, DataClaw provides a less data constrained diagnostic testbed for probing the capability boundaries of autonomous data-analysis agents1 .
