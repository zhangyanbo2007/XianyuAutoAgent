中文对照翻译

您的智能体也在老化：部署系统的智能体生命周期工程。您的智能体一站式老化护理：https://AgingBench.github.io/。长寿命 AI 智能体越来越多地作为持久运行系统部署，但它们仍然像刚初始化的模型一样被评估。部署首日基准测试遗漏了一个基本的系统问题：智能体部署后能保持多长时间的可靠性？即使模型权重被冻结，智能体的有效状态也会随着它压缩交互历史、从不断增长的内存存储中检索、在更新后修正事实以及经历例行维护而持续变化。因此，可靠性成为完整智能体框架的生命周期属性，而不仅仅是基础模型的快照属性。我们引入了 AgingBench，一个用于智能体生命周期工程的纵向可靠性基准：它不仅衡量部署的智能体是否会性能下降，还衡量性能下降的形式以及修复应针对何处。AgingBench 将智能体老化分为四种机制：压缩老化，即写入时摘要遗漏了未来相关细节；干扰老化，即累积的相似记忆挤占了目标事实；修正老化，即更改或派生状态未正确更新；以及维护老化，即清理或重新压缩等生命周期事件触发了退化。为了诊断这些故障，AgingBench 使用时间依赖图和配对反事实探针，为内存管道的写入、检索和利用阶段生成诊断配置文件。在 7 种场景、14 种模型、多种内存策略以及运行器控制和自主智能体上，超过约 400 次运行（跨越 8 到 200 个会话）表明，智能体老化不是一维的：行为测试可能保持良好，而事实准确性却下降；派生状态跟踪在单个模型中可能急剧崩溃；并且相同的错误答案可能需要不同的修复，具体取决于诊断配置文件指向的内容。这些结果表明，可靠的智能体部署需要生命周期评估、机制级诊断和阶段性修复，而不仅仅是更强大的部署首日模型。

原始摘要

 Your One-Stop Aging Care: https://AgingBench.github.io/ Long-lived AI agents are increasingly deployed as persistent operational systems, yet they are still evaluated like freshly initialized models. Day-one benchmarks miss a basic systems question: how long does an agent remain reliable after deployment? Even when model weights are frozen, an agent’s effective state keeps changing as it compresses interaction history, retrieves from a growing memory store, revises facts after updates, and undergoes routine maintenance. Reliability therefore becomes a lifespan property of the full agent harness, not only a snapshot property of the base model. We introduce AgingBench, a longitudinal reliability benchmark for agent lifespan engineering: measuring not only whether deployed agents degrade, but what form the degradation takes and where repair should target. AgingBench organizes agent aging into four mechanisms: compression aging, where write-time summarization drops future-relevant details; interference aging, where accumulated similar memories crowd out the target fact; revision aging, where changed or derived state is not updated correctly; and maintenance aging, where lifecycle events such as flushing or recompaction trigger regressions. To diagnose these failures, AgingBench uses temporal dependency graphs and paired counterfactual probes that produce diagnostic profiles for the write, retrieval, and utilization stages of the memory pipeline. Across 7 scenarios, 14 models, multiple memory policies, and both runner-controlled and autonomous agents, over ∼400 runs spanning 8 - 200 sessions show that agent aging is not one-dimensional: behavioral tests can remain clean while factual precision decays; derived-state tracking can collapse sharply within a single model; and the same wrong answer can require different repairs depending on what the diagnostic profile points to. These results suggest that reliable agent deployment requires lifespan evaluation, mechanism-level diagnosis, and stage-targeted repair, not only stronger day-one models.
