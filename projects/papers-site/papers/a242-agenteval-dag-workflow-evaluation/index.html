中文对照翻译

将推理、工具使用和合成串联成多步骤工作流程的智能体系统正在投入生产，然而，普遍的评估实践，如端到端结果检查和临时跟踪检查，系统性地掩盖了在实际错误预算中占主导地位的中间故障。我们提出了AGENT E VAL，一个将智能体执行形式化为评估有向无环图（DAG）的框架，其中每个节点都带有由经过校准的LLM评判者（GPT-4o）评估的类型化质量指标，通过分层故障分类法（3个级别，21个子类别）进行分类，并链接到上游依赖项以实现自动化根本原因归因。一项消融研究分离了基于DAG的依赖建模的影响：与使用相同评判者和评估标准的平面步骤级评估相比，它单独对故障检测召回率贡献了+22个百分点，对根本原因准确性贡献了+34个百分点。在三个生产工作流程中（450个测试用例，两个智能体模型家族，主要是顺序架构，非DAG跟踪率为12%），AGENT E VAL的故障检测召回率比端到端评估高2.17倍（0.89 vs. 0.41），与人类专家的Cohen’s κ一致性为0.84，根本原因准确率为72%，而人类上限为81%。对τ-bench和SWEbench跟踪的跨系统评估证实了其可迁移性（故障检测召回率≥0.78），无需修改分类法或评估标准。一项为期4个月、有18名工程师参与的试点项目通过CI/CD集成的回归测试检测到23个预发布回归问题，将中位根本原因识别时间从4.2小时减少到22分钟，并推动了两个工作流程中可衡量的故障率降低。

原始摘要

Agentic systems that chain reasoning, tool use, and synthesis into multi-step workflows are entering production, yet prevailing evaluation practices like end-to-end outcome checks and ad-hoc trace inspection systematically mask the intermediate failures that dominate realworld error budgets. We present AGENT E VAL, a framework that formalizes agent executions as evaluation directed acyclic graphs (DAGs), where each node carries typed quality metrics assessed by a calibrated LLM judge (GPT-4o), classified through a hierarchical failure taxonomy (3 levels, 21 subcategories), and linked to upstream dependencies for automated root cause attribution. An ablation study isolates the impact of DAG-based dependency modeling: it alone contributes +22 percentage points to failure detection recall and +34 pp to root cause accuracy over flat step-level evaluation with identical judges and rubrics. Across three production workflows (450 test cases, two agent model families, predominantly sequential architectures with a 12% nonDAG trace rate), AGENT E VAL achieves 2.17× higher failure detection recall than end-to-end evaluation (0.89 vs. 0.41), Cohen’s κ = 0.84 agreement with human experts, and 72% root cause accuracy against an 81% human ceiling. Cross-system evaluation on τ -bench and SWEbench traces confirms transferability (failure detection recall ≥ 0.78) without taxonomy or rubric modification. A 4-month pilot with 18 engineers detected 23 pre-release regressions through CI/CD-integrated regression testing, reducing median root-cause identification time from 4.2 hours to 22 minutes and driving measurable failure rate reductions in two workflows.
