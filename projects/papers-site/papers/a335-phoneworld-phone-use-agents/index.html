核心要点

问题/背景
手机使用 agent 的瓶颈不是缺少单个 benchmark，而是缺少可规模化构造、可复现、可验证的 phone-use environments。
方法/机制
PhoneWorld 把真实 GUI trajectories 和 screenshots 转换为可控环境、可执行任务和自动 verifier，目标是让 phone-use agent 的训练和评测可以持续扩展。
结果/证据
这类环境生成管线比单一任务集更重要，因为 computer-use / mobile-use agent 需要不断产生新的可验证任务。
收录价值
它值得收录，因为它提供了面向移动 GUI agent 的环境生产基础设施，符合本库对 agent harness 和 verifiable training environments 的关注。
完整收录解读

手机使用 agent 的瓶颈不是缺少单个 benchmark，而是缺少可规模化构造、可复现、可验证的 phone-use environments。

PhoneWorld 把真实 GUI trajectories 和 screenshots 转换为可控环境、可执行任务和自动 verifier，目标是让 phone-use agent 的训练和评测可以持续扩展。

这类环境生成管线比单一任务集更重要，因为 computer-use / mobile-use agent 需要不断产生新的可验证任务。

它值得收录，因为它提供了面向移动 GUI agent 的环境生产基础设施，符合本库对 agent harness 和 verifiable training environments 的关注。

原始摘要与中文对照

中文对照翻译

PhoneWorld 引入了一个可重用的流水线，可以将真实的移动设备GUI轨迹和截图转换为可控的手机使用环境、可执行的任务、自动验证器以及可扩展的智能体训练/评估设置。

原始摘要

PhoneWorld introduces a reusable pipeline that converts real mobile GUI trajectories and screenshots into controllable phone-use environments, executable tasks, automatic verifiers, and scalable agent training/evaluation settings.
