原始摘要与中文对照

中文对照翻译

随着模型家族、训练方案和计算预算日益标准化，机器学习系统的进一步提升越来越依赖于数据。然而，数据工程在很大程度上仍然是手动和临时的：实践者反复搜索外部数据集，将其适应现有管道，通过下游训练验证候选数据，并从先前的尝试中吸取经验。我们研究了任务条件下的自主数据工程，其中自主智能体通过仅优化数据侧来改进固定的学习算法，包括外部数据发现、数据选择和组合、清洗和转换。目标是在不改变学习算法的情况下获得更强大的下游解决方案。为了解决自主数据工程中固有的开放式搜索空间、分支依赖的细化和延迟验证问题，我们提出了DataMaster，这是一个数据智能体框架，它集成了树形结构搜索、共享候选数据和累积记忆。DataMaster由三个关键组件组成：一个组织替代数据工程分支的DataTree，一个存储已发现的外部数据源以供重用的共享Data Pool，以及一个记录节点结果、工件和可重用发现的Global Memory。这些组件共同使智能体能够发现候选数据，构建可执行的训练输入，通过下游反馈对其进行评估，并将有用的证据跨分支传递。我们在两种类型的基准测试MLEBench Lite和PostTrainBench上评估了DataMaster。在MLE-Bench Lite上，它将奖牌率比初始分数提高了32.27%；在PostTrainBench上，它在GPQA上超越了指令模型（31.02% 对 30.35%）。

原始摘要

As model families, training recipes, and compute budgets become increasingly standardized, further gains in machine learning systems depend increasingly on data. Yet data engineering remains largely manual and ad hoc: practitioners repeatedly search for external datasets, adapt them to existing pipelines, validate candidate data through downstream training, and carry forward lessons from prior attempts. We study task-conditioned autonomous data engineering, where an autonomous agent improves a fixed learning algorithm by optimizing only the data side, including external data discovery, data selection and composition, cleaning and transformation. The goal is to obtain a stronger downstream solution while leaving the learning algorithm unchanged. To address the open-ended search space, branch-dependent refinement, and delayed validation inherent in autonomous data engineering, we propose DataMaster, a data-agent framework that integrates tree-structured search, shared candidate data, and cumulative memory. DataMaster consists of three key components: a DataTree that organizes alternative data-engineering branches, a shared Data Pool that stores discovered external data sources for reuse, and a Global Memory that records node outcomes, artifacts, and reusable findings. Together, these components allow the agent to discover candidate data, construct executable training inputs, evaluate them through downstream feedback, and carry useful evidence across branches. We evaluate DataMaster on two types of benchmarks, MLEBench Lite and PostTrainBench. On MLE-Bench Lite, it improves medal rate by 32.27% over the initial score; on PostTrainBench, it surpasses the instruct model on GPQA (31.02% vs 30.35%).
