核心要点

问题/背景
随着 agent 系统越来越多地自动打开 URL、读取网页预览、调用外部工具，安全风险不再只发生在最终可见回答里。系统可能在用户看不到的中间步骤里发出外联请求，把敏感上下文悄悄带出运行环境，而现有输出级 safety checks 往往根本察觉不到。
方法/机制
Silent Egress 针对这一点提出 implicit prompt injection 的系统级风险图景：恶意网页可以把攻击藏在自动生成的标题、metadata 和 snippets 里，诱导 agent 触发 outbound requests 并外泄敏感信息。论文进一步提出 sharded exfiltration，把泄露内容拆成多次请求以绕过简单 DLP，并用实验表明 prompt-layer 防御保护有限，而 domain allowlisting、redirect-chain analysis 等 system/network controls 更有效。
结果/证据
这篇工作值得收录，因为它把网络外联本身提升为 agent security 的 first-class outcome。与很多只盯回答文本的安全论文不同，这篇工作把关注点推到 runtime 和 network boundary，对真实部署的 agent sandbox、egress control 和 provenance tracking 都有直接启发。
收录价值
它没有升到更高一级，是因为当前主要是单条风险线上的强系统安全论文，虽然问题切得很准，但还没有像更高阶工作那样给出统一的 agent runtime security architecture。它非常值得收，但暂时仍属于强突破级。

静默泄露：当隐式提示注入使LLM代理无痕泄露时 摘要 代理式大型语言模型系统越来越多地通过检索URL和调用外部工具来自动化任务。我们发现这种工作流程会产生隐式提示注入：嵌入在自动生成的URL预览（包括标题、元数据和摘要）中的对抗性指令，可能引入我们称之为静默泄露的系统级风险。利用一个完全本地化且可复现的测试平台，我们证明恶意网页可以诱导代理发出窃取敏感运行时上下文的出站请求，即使最终向用户显示的响应看起来无害。在使用基于qwen2.5:7b的代理进行的480次实验运行中，攻击以高概率成功（P (泄露) ≈ 0.89），并且95%的成功攻击未被基于输出的安全检查检测到。我们还引入了分片窃取，其中敏感信息被拆分到多个请求中以避免检测。这种策略将单请求泄露指标降低了73% (Leak@1)，并绕过了简单的数据丢失防护机制。我们的消融结果表明，应用于提示层的防御提供有限的保护，而系统和网络层的控制，例如域名白名单和重定向链分析，则显著更有效。这些发现表明，在代理式LLM系统中，网络出站应被视为一级安全结果。我们概述了超越提示层加固的架构方向，包括溯源跟踪和能力隔离。关键词 LLM安全 · 代理系统 · 提示注入 · 数据窃取 · 网络出站

原始摘要

A BSTRACT Agentic large language model systems increasingly automate tasks by retrieving URLs and calling external tools. We show that this workflow gives rise to implicit prompt injection: adversarial instructions embedded in automatically generated URL previews, including titles, metadata, and snippets, can introduce a system-level risk that we refer to as silent egress. Using a fully local and reproducible testbed, we demonstrate that a malicious web page can induce an agent to issue outbound requests that exfiltrate sensitive runtime context, even when the final response shown to the user appears harmless. In 480 experimental runs with a qwen2.5:7b-based agent, the attack succeeds with high probability (P (egress) ≈ 0.89), and 95% of successful attacks are not detected by output-based safety checks. We also introduce sharded exfiltration, where sensitive information is split across multiple requests to avoid detection. This strategy reduces single-request leakage metrics by 73% (Leak@1) and bypasses simple data loss prevention mechanisms. Our ablation results indicate that defenses applied at the prompt layer offer limited protection, while controls at the system and network layers, such as domain allowlisting and redirect-chain analysis, are considerably more effective. These findings suggest that network egress should be treated as a first-class security outcome in agentic LLM systems. We outline architectural directions, including provenance tracking and capability isolation, that go beyond prompt-level hardening. Keywords LLM security · Agentic Systems · Prompt Injection · Data Exfiltration · Network Egress
