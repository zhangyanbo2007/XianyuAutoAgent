原始摘要与中文对照

中文对照翻译

科学算法发现是一个迭代过程：提出假设、实现、压力测试并修订。当前的LLM引导搜索系统加速了提案生成，但通常通过优化仅包含代码的工件，并辅以薄弱的正确性/原创性门控，从而未能充分体现科学结构。我们提出了CliffSearch，一个代理进化框架，其中核心进化操作符（配对选择、交叉、变异和审查）被实现为LLM代理，并且循环围绕三个原则设计：（1）每个节点都是一个结构化的科学工件，以理论+代码或仅代码模式实例化；（2）审稿人对正确性和原创性的判断是与优化感兴趣的基准指标并列的一流选择门；（3）变异分为探索和纠正路径，具有不同的目标。探索变异从相邻科学领域引入思想以增加新颖性，而纠正变异则利用审稿人对理论、代码、基准结果和运行时错误的信号进行有针对性的证据引导修复。即使在仅代码模式下，节点仍然包含一个必需的summary_md字段，记录所提出的设计原则，因此该模式抑制了形式化的theory_content，而不是完全消除构思；实际上，这也为可执行代码释放了更多的代理token预算。我们通过三项基于基准的研究来阐述该框架：transformer超连接进化、在固定nanoGPT堆栈上的优化器发现，以及一个较小的原生优化器消融研究。在这些设置中，相同的循环支持明确的指标方向、可重现的持久性，以及在受控搜索条件下对发现进行审稿人门控比较。结果是一个发现工作流，它在受控新颖性约束下优化任务指标，同时优先考虑科学可解释性和正确性，而不是仅仅最大化候选吞吐量。完整的运行工件、交互式可视化以及报告研究的最佳导出节点可在https://cliffsearch.ai获取。

原始摘要

Scientific algorithm discovery is iterative: hypotheses are proposed, implemented, stresstested, and revised. Current LLM-guided search systems accelerate proposal generation, but often under-represent scientific structure by optimizing code-only artifacts with weak correctness/originality gating. We present CliffSearch, an agentic evolutionary framework in which the core evolution operators (pair selection, crossover, mutation, and review) are implemented as LLM agents, and the loop is designed around three principles: (1) each node is a structured scientific artifact, instantiated in either theory+code or code_only mode, (2) reviewer judgments of correctness and originality are first-class selection gates alongside optimization of the benchmark metric of interest, and (3) mutation is split into exploration and correction pathways with distinct objectives. Exploration mutation imports ideas from adjacent scientific domains to increase novelty, while correction mutation performs targeted evidence-guided repair using reviewer signals over theory, code, benchmark results, and runtime errors. Even in code_only mode, nodes still carry a required summary_md field that records the design principles being proposed, so the mode suppresses formal theory_content rather than eliminating ideation altogether; in practice this also frees more of the agent token budget for executable code. We illustrate the framework on three benchmark-grounded studies: transformer hyper-connection evolution, optimizer discovery on a fixed nanoGPT stack, and a smaller native-optimizer ablation. Across these settings, the same loop supports explicit metric direction, reproducible persistence, and reviewer-gated comparison of discoveries under controlled search conditions. The result is a discovery workflow that prioritizes scientific interpretability and correctness while optimizing task metrics under controlled novelty constraints, rather than maximizing candidate throughput alone. Full run artifacts, interactive visualizations, and exported best nodes for the reported studies are available at https://cliffsearch.ai.
