中文对照翻译

模型上下文协议（MCP）正越来越多地被工具集成型LLM智能体采用，但其多层设计和第三方服务器生态系统扩大了工具元数据、不可信输出、跨工具流、多模态输入和供应链向量等方面的风险。现有的MCP基准测试主要衡量对恶意输入的鲁棒性，但提供的修复指导有限。我们提出了MCP陷阱实验室，这是一个协议感知的安全测试框架，它将开发者陷阱操作化为可重现的场景，并使用MCP跟踪和客观验证器（而非智能体自报告）来验证结果。我们通过六种服务器变体（基线/强化）实例化了三个工作流挑战（电子邮件、文档、加密），并在统一的、基于跟踪的评估中建模了三种攻击家族——工具元数据投毒、傀儡服务器和多模态图像到工具链。在对六种变体（36个二元标签）进行的Tier-1静态分析中，我们的分析器在四个可静态检查的陷阱类别（P1/P2/P5/P6）上实现了F1=1.0，并将跨工具转发和图像到工具泄露（P3/P4）标记为依赖于跟踪/数据流。应用推荐的强化措施消除了所有Tier-1发现（29→0），并以平均27行代码（LOC）的成本降低了框架风险评分（10.0→0.0）。最后，在来自电子邮件系统挑战（工具投毒+傀儡）的初步19次运行语料库中，智能体叙述与跟踪证据在63.2%的运行中以及100%的汇聚动作运行中存在分歧，这促使了基于跟踪的审计和回归测试。总之，陷阱实验室能够在现实多向量条件下对MCP工具服务器进行实用的端到端评估和强化。

原始摘要

Model Context Protocol (MCP) is increasingly adopted for tool-integrated LLM agents, but its multi-layer design and third-party server ecosystem expand risks across tool metadata, untrusted outputs, cross-tool flows, multimodal inputs, and supply-chain vectors. Existing MCP benchmarks largely measure robustness to malicious inputs but offer limited remediation guidance. We present MCP Pitfall Lab, a protocol-aware security testing framework that operationalizes developer pitfalls as reproducible scenarios and validates outcomes with MCP traces and objective validators (rather than agent self-report). We instantiate three workflow challenges (email, document, crypto) with six server variants (baseline/hardened) and model three attack families—toolmetadata poisoning, puppet servers, and multimodal image-totool chains—in a unified, trace-grounded evaluation. In Tier-1 static analysis over six variants (36 binary labels), our analyzer achieves F1=1.0 on four statically checkable pitfall classes (P1/P2/P5/P6) and flags cross-tool forwarding and imageto-tool leakage (P3/P4) as trace/dataflow-dependent. Applying recommended hardening eliminates all Tier-1 findings (29→0) and reduces the framework risk score (10.0→0.0) at a mean cost of 27 lines of code (LOC). Finally, in a preliminary 19-run corpus from the emailsystem challenge (tool poisoning + puppet), agent narratives diverge from trace evidence in 63.2% of runs and 100% of sink-action runs, motivating trace-based auditing and regression testing. Overall, Pitfall Lab enables practical, end-to-end assessment and hardening of MCP tool servers under realistic multi-vector conditions.
