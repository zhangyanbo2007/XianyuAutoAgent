大型语言模型（LLM）智能体越来越依赖可重用技能：结合指令、控制流、约束和工具调用的能力包。然而，在当前的智能体系统中，技能仍然以文本密集型工件的形式表示，主要是SKILL.md风格的文档，其机器可用的证据主要嵌入在自然语言描述中。因此，以技能为中心的智能体系统面临一个表示问题：管理技能集合和在智能体执行期间使用技能都需要对调用接口、执行结构和具体副作用进行推理，但这些信号通常纠缠在单一文本表面中。因此，技能知识的显式表示可能有助于使这些工件更容易被机器获取和利用。借鉴Schank和Abelson关于认知语言学表示的经典著作中的记忆组织包、脚本理论和概念依赖，我们引入了据我们所知，第一个用于智能体技能工件的结构化表示，它解耦了技能层面的调度信号、场景层面的执行结构和逻辑层面的动作/资源使用证据：调度-结构-逻辑（SSL）表示。我们使用基于LLM的规范器实例化SSL，并在技能发现和风险评估两项任务中评估了SSL派生的表示。实验表明，SSL显著优于纯文本基线：在技能发现中，MRR@50从0.649提高到0.729；在风险评估中，宏F1从0.409提高到0.509。这些发现表明，显式、源头扎根的结构可以使智能体技能更容易搜索和审查，将SSL定位为迈向更可检查、可重用和可操作的技能表示的实用步骤，而非成熟标准或端到端技能管理机制。

原始摘要

Large language model (LLM) agents increasingly rely on reusable skills: capability packages that combine instructions, control flow, constraints, and tool calls. In current agent systems, however, skills are still represented by text-heavy artifacts, mainly SKILL.md-style documents whose machine-usable evidence remains embedded largely in natural-language descriptions. As a result, skill-centered agent systems face a representation problem: both managing skill collections and using skills during agent execution require reasoning over invocation interfaces, execution structure, and concrete side effects, but these signals are often entangled in a single textual surface. An explicit representation of skill knowledge may therefore help make these artifacts easier for machines to acquire and leverage. Drawing on Memory Organization Packets, Script Theory, and Conceptual Dependency from Schank and Abelson’s classical work on cognitive linguistic representation, we introduce what is, to our knowledge, the first structured representation for agent skill artifacts that disentangles skill-level scheduling signals, scene-level execution structure, and logic-level action/resource-use evidence: the Scheduling-StructuralLogical (SSL) representation. We instantiate SSL with an LLM-based normalizer and evaluate SSL-derived representations in two tasks, Skill Discovery and Risk Assessment. The experiment shows that SSL significantly outperforms the textonly baselines: in Skill Discovery, MRR@50 improves from 0.649 to 0.729; in Risk Assessment, macro F1 improves from 0.409 to 0.509. These findings suggest that an explicit, source-grounded structure can make agent skills easier to search and review, positioning SSL as a practical step toward more inspectable, reusable, and operationally actionable skill representations, rather than a finished standard or end-to-end skill-management mechanism. 1
