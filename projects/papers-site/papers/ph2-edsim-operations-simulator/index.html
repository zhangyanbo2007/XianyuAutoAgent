EDSim: An Agentic Simulator for Emergency Department Operations

公共卫生与医疗运营 · 发表：2026-03-03

发表时间
2026-03-03

核心要点

问题/背景
急诊科运营长期受拥堵、床位分配和人力调度约束影响。传统离散事件模拟或普通 agent-based simulation 能对齐宏观等待时间和吞吐量，但很难表达患者与医护之间的微观交互、临场沟通和状态依赖决策，而这些恰恰会改变瓶颈位置和流程效率。
方法/机制
EDSim 提出的是一种 agentic operations simulator：患者与医护由大语言模型代理驱动，但这些代理并不是完全自由对话，而是被临床规则、急诊全局状态和模块化 patient journey 结构所约束。这样既保留了微观行为的可解释性，又能让运营方快速跑 what-if 场景，如重新分配床位或人员。
结果/证据
这项工作的仓库价值在于它把 LLM agent 从“做预测/问答”推进到“做 operations simulation”。对公共卫生与医疗运营这条线来说，这是一种更接近真实 workflow 的新模式：既能复现流转过程，又能让管理者测试干预方案。
收录价值
它暂时仍是 breakthrough，因为当前证据主要是对历史等待时间分布的对齐和少量场景实验，外部医院验证、长期稳定性和模型治理问题都还没有完全解决。Research Square 预印本的状态也意味着后续仍需持续跟踪。
完整收录解读

急诊科运营长期受拥堵、床位分配和人力调度约束影响。传统离散事件模拟或普通 agent-based simulation 能对齐宏观等待时间和吞吐量，但很难表达患者与医护之间的微观交互、临场沟通和状态依赖决策，而这些恰恰会改变瓶颈位置和流程效率。

EDSim 提出的是一种 agentic operations simulator：患者与医护由大语言模型代理驱动，但这些代理并不是完全自由对话，而是被临床规则、急诊全局状态和模块化 patient journey 结构所约束。这样既保留了微观行为的可解释性，又能让运营方快速跑 what-if 场景，如重新分配床位或人员。

这项工作的仓库价值在于它把 LLM agent 从“做预测/问答”推进到“做 operations simulation”。对公共卫生与医疗运营这条线来说，这是一种更接近真实 workflow 的新模式：既能复现流转过程，又能让管理者测试干预方案。

它暂时仍是 breakthrough，因为当前证据主要是对历史等待时间分布的对齐和少量场景实验，外部医院验证、长期稳定性和模型治理问题都还没有完全解决。Research Square 预印本的状态也意味着后续仍需持续跟踪。
