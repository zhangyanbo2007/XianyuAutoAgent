核心要点

问题/背景
这篇论文抓住了一个经常被低估但越来越关键的问题：agent 表现高度依赖 harness engineering，但 harness 往往埋在控制器代码、运行时约定和框架细节里，难以迁移、比较和系统研究。结果是很多 agent 进步其实发生在 harness 层，却没有形成稳定的可复用对象。
方法/机制
作者提出 Natural-Language Agent Harnesses，把高层控制逻辑外化成可编辑的自然语言工件，并配套 Intelligent Harness Runtime，通过显式 contract、durable artifact 和轻量 adapter 来执行这些 harness。真正的新意不只是“用自然语言写配置”，而是把 harness 从隐藏代码路径提升为可移植、可实验、可迁移的独立执行层。
结果/证据
这条路线对本仓库很重要，因为它触及 agent 系统工程里最缺少标准化的一层：控制逻辑、运行时约束和能力编排接口。如果 harness 可以像 prompt、tool schema 或 memory policy 一样被抽象出来，后续 agent 复现、迁移和比较都会更清晰。

原始摘要与中文对照

中文对照翻译

标题：自然语言智能体线束。智能体性能日益依赖于线束工程，然而线束设计通常隐藏在控制器代码和运行时特定约定中，这使得其难以作为科学对象进行迁移、比较和研究。我们探讨智能体线束的高级控制逻辑是否可以外部化为可移植的可执行工件。我们引入了自然语言智能体线束（NLAHs），它以可编辑的自然语言表达线束行为，以及智能线束运行时（IHR），这是一个共享运行时，通过明确的契约、持久工件和轻量级适配器来执行这些线束。在编码和计算机使用基准测试中，我们对操作可行性、模块消融以及代码到文本的线束迁移进行了受控评估。

原始摘要

Agent performance increasingly depends on harness engineering, yet harness design is usually buried in controller code and runtimespecific conventions, making it hard to transfer, compare, and study as a scientific object. We ask whether the high-level control logic of an agent harness can instead be externalized as a portable executable artifact. We introduce Natural-Language Agent Harnesses (NLAHs), which express harness behavior in editable natural language, and Intelligent Harness Runtime (IHR), a shared runtime that executes these harnesses through explicit contracts, durable artifacts, and lightweight adapters. Across coding and computer-use benchmarks, we conduct controlled evaluations of operational viability, module ablation, and code-to-text harness migration.
