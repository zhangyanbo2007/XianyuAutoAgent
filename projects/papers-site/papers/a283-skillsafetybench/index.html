中文对照翻译

仅在用户指令和固定工具接口的条件下，智能体可能会读取并执行技能文件、辅助脚本、运行时配置、检索资源以及其他本地工件（Xu and Yan, 2026; Ling et al., 2026; Schmotz et al., 2026）。因此，技能加载成为一个与安全相关的输入通道：非用户编写的材料可以影响智能体如何解释、规划和执行良性请求（Greshake et al., 2023; Liu et al., 2024; Zhan et al., 2024; Debenedetti et al., 2024）。我们研究了由此产生的技能介导的本地非用户攻击面类别。在这种设置中，用户发出一个良性请求，但智能体依赖于一个面向任务的技能，其指令和相关本地资源已被对抗性地塑造，以将执行重定向到不安全行为（Yi et al., 2025; Liu et al., 2023b）。此类攻击利用了技能不仅仅是提示的事实：它们将程序、文件、脚本、内存和依赖项组织成一个看似合法的 workflow。因此，恶意影响可能分布在技能指导、包装脚本、内存条目、本地语料库、依赖工件或其他技能导致智能体查阅的材料中（Zou et al., 2025; RoyChowdhury et al., 2024）。核心安全问题不仅在于智能体是否拒绝明确的恶意指令，还在于当技能介导的本地上下文作为权威的 workflow 支持呈现时，它是否保持安全（Chen et al., 2025）。当前的评估仅部分涵盖了这种设置。现实的智能体基准在可执行、有状态或应用级环境中衡量任务完成度（Liu et al., 2023a; Zhou et al., 2023; Xie et al., 2024; Jimenez et al., 2023; Yao et al., 2024; Lu et al., 2025; Trivedi et al., 2024）；安全基准研究工具滥用、网络智能体攻击、有害智能体行为或对抗性智能体交互循环（Ruan et al., 2023; Debenedetti et al., 2024; Zhang et al., 2024; Evtimov et al., 2025; Andriushchenko et al., 2024）；而技能安全工作。可重用技能正成为扩展大型语言模型智能体的常见接口，它将程序指导与文件、工具、内存和执行环境的访问打包在一起。然而，这种模块化引入了现有安全评估在很大程度上忽略的攻击面：即使用户请求是良性的，与任务相关的技能材料或本地工件也可能将智能体引向不安全的操作。我们提出了 SkillSafetyBench，一个可运行的基准，用于评估此类技能介导的安全故障。SkillSafetyBench 包含 47 个任务中的 155 个对抗性案例。

Skill loading therefore becomes a safety-relevant input channel: non-user-authored materials can influence how an agent interprets, plans, and executes a benign request (Greshake et al., 2023; Liu et al., 2024; Zhan et al., 2024; Debenedetti et al., 2024). We study the resulting class of skill-mediated local non-user attack surfaces. In this setting, a user issues a benign request, but the agent relies on a task-facing skill whose instructions and associated local resources have been adversarially shaped to redirect execution toward unsafe behavior (Yi et al., 2025; Liu et al., 2023b). Such attacks exploit the fact that skills are not merely prompts: they organize procedures, files, scripts, memory, and dependencies into an apparently legitimate workflow. Malicious influence may therefore be distributed across skill guidance, wrapper scripts, memory entries, local corpora, dependency artifacts, or other materials that the skill causes the agent to consult (Zou et al., 2025; RoyChowdhury et al., 2024). The central safety question is not only whether an agent rejects an explicitly malicious instruction, but whether it remains safe when skill-mediated local context is presented as authoritative workflow support (Chen et al., 2025). Current evaluations only partially cover this setting. Realistic agent benchmarks measure task completion in executable, stateful, or application-level environments (Liu et al., 2023a; Zhou et al., 2023; Xie et al., 2024; Jimenez et al., 2023; Yao et al., 2024; Lu et al., 2025; Trivedi et al., 2024); security benchmarks study tool misuse, web-agent attacks, harmful agent behavior, or adversarial agent interaction loops (Ruan et al., 2023; Debenedetti et al., 2024; Zhang et al., 2024; Evtimov et al., 2025; Andriushchenko et al., 2024); and skill-security work Reusable skills are becoming a common interface for extending large language model agents, packaging procedural guidance with access to files, tools, memory, and execution environments. However, this modularity introduces attack surfaces that are largely missed by existing safety evaluations: even when the user request is benign, task-relevant skill materials or local artifacts can steer an agent toward unsafe actions.
