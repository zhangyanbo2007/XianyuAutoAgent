HERMES: KV Cache as Hierarchical Memory for Efficient Streaming Video Understanding

核心要点

问题/背景
HERMES 针对多模态大模型从离线视频理解走向 streaming video understanding 的核心矛盾：需要稳定理解、实时响应和低 GPU memory overhead，但常规长视频处理会随着帧数增长导致 TTFT 和显存不可控。
方法/机制
论文基于 layer-wise attention preference 的机制观察，把 KV cache 重新定义为包含多粒度视频信息的 hierarchical memory，而不是一次性缓存或简单 token eviction。
结果/证据
推理时 HERMES 复用 compact KV cache，并用分层管理策略保留不同粒度视频 token；用户查询到达时不需要额外 retrieval 或外部 memory computation，从而保证连续视频流交互的实时响应。
收录价值
它值得正式收录，因为它把 streaming video MLLM 的效率问题转化为 KV-cache-as-memory 的系统接口，连接长视频理解、实时多模态交互、KV cache 管理和训练无关部署优化。
完整收录解读

HERMES 针对多模态大模型从离线视频理解走向 streaming video understanding 的核心矛盾：需要稳定理解、实时响应和低 GPU memory overhead，但常规长视频处理会随着帧数增长导致 TTFT 和显存不可控。

论文基于 layer-wise attention preference 的机制观察，把 KV cache 重新定义为包含多粒度视频信息的 hierarchical memory，而不是一次性缓存或简单 token eviction。

推理时 HERMES 复用 compact KV cache，并用分层管理策略保留不同粒度视频 token；用户查询到达时不需要额外 retrieval 或外部 memory computation，从而保证连续视频流交互的实时响应。

它值得正式收录，因为它把 streaming video MLLM 的效率问题转化为 KV-cache-as-memory 的系统接口，连接长视频理解、实时多模态交互、KV cache 管理和训练无关部署优化。

原始摘要与中文对照

中文对照翻译

HERMES 是一种训练无需架构，用于高效的流式视频理解。它将 KV 缓存视为视频信息在多个粒度上的分层记忆，在推理过程中重用紧凑的缓存，并报告了高达 10 倍的 TTFT 速度，同时减少高达 68% 的视频标记数量，同时保持或提高准确性。

原始摘要

HERMES is a training-free architecture for efficient streaming video understanding. It treats KV cache as hierarchical memory over video information at multiple granularities, reuses compact cache during inference, and reports 10x faster TTFT with up to 68% fewer video tokens while maintaining or improving accuracy.
