原始摘要与中文对照

中文对照翻译

工具调用是智能体系统的核心能力，然而，失败往往并非源于单个工具调用，而是源于多个工具如何协同组织和执行。现有方法将工具执行与逐步语言推理或显式规划紧密耦合，导致行为脆弱且执行开销高昂。为克服这些局限性，我们从工具编排的角度重新审视工具调用。我们的关键见解是，有效的编排不需要精确的依赖图或细粒度规划。相反，粗粒度分层结构足以提供全局指导，而执行时错误可以在局部进行纠正。具体而言，我们将工具编排建模为学习一种分层执行结构，该结构捕获高层工具依赖关系，并通过上下文约束诱导层级执行。为处理执行时故障，我们引入了一种模式感知的反射式纠正机制，该机制能局部检测并修复错误。这种设计将错误限制在单个工具调用中，并避免重新规划整个执行轨迹。这种结构化执行范式为智能体系统提供了一个轻量级且可重用的编排组件。实验结果表明，我们的方法实现了鲁棒的工具执行，同时降低了执行复杂性和开销。代码将公开发布。

原始摘要

Tool invocation is a core capability of agentic systems, yet failures often arise not from individual tool calls but from how multiple tools are organized and executed together. Existing approaches tightly couple tool execution with stepwise language reasoning or explicit planning, leading to brittle behavior and high execution overhead. To overcome these limitations, we revisit tool invocation from the perspective of tool orchestration. Our key insight is that effective orchestration does not require precise dependency graphs or fine-grained planning. Instead, a coarsegrained layer structure suffices to provide global guidance, while execution-time errors can be corrected locally. Specifically, we model tool orchestration as learning a layered execution structure that captures high-level tool dependencies, inducing layer-wise execution through context constraints. To handle execution-time failures, we introduce a schema-aware reflective correction mechanism that detects and repairs errors locally. This design confines errors to individual tool calls and avoids re-planning entire execution trajectories. This structured execution paradigm enables a lightweight and reusable orchestration component for agentic systems. Experimental results show that our approach achieves robust tool execution while reducing execution complexity and overhead. Code will be made publicly available.
